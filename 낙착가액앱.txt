[최종 맞춤형 앱 기획서]
서울시 토목·포장 공사 전용 나라장터 입찰 전략 보조 앱

==================================================
1. 앱 목적
==================================================

이 앱은 비드큐를 사용하지 않는다.

조달청 나라장터 공공데이터 API에서 실시간 입찰공고 정보를 받아오고,
그중 서울시 관공서에서 발주하는 토목공사와 포장공사 입찰만 필터링한다.

현재 입찰공고 정보와 과거 유사 공사 낙찰 결과를 비교 분석하여,
사용자가 조달청 입찰에 넣을 최종 투찰 금액을 결정할 수 있도록 돕는
모바일 카드형 입찰 전략 보조 앱이다.

이 앱은 “낙찰 보장 앱”이 아니다.
정확한 정의는 “나라장터 공공데이터 기반 투찰 전략 시뮬레이터”이다.

==================================================
2. 핵심 구조
==================================================

나라장터 입찰공고 API
→ 서울시 관공서 공사만 필터링
→ 토목·포장 관련 공고만 필터링
→ 공고 상세정보 정리
→ 나라장터 낙찰정보 API로 과거 유사 공사 수집
→ 유사 공사 낙찰률·사정률 패턴 분석
→ 보수형 / 중간형 / 공격형 투찰 후보 금액 계산
→ 사용자가 최종 투찰 금액 선택
→ 카드형 결과 대시보드 출력

9. 투찰 후보 금액 산출 방식
==================================================

AI는 낙찰 정답을 찍지 않는다.

대신 과거 유사 공사의 패턴을 바탕으로 3가지 후보를 계산한다.

--------------------------------------------------

[보수형]
낮은 사정률 기준

목적:
무리하지 않는 안정형 금액 확인

예:
사정률 99.94%

--------------------------------------------------

[중간형]
유사 공사 평균 또는 중앙값 기준

목적:
가장 현실적인 기준 금액 확인

예:
사정률 99.98%

--------------------------------------------------

[공격형]
최근 집중 구간 상단 기준

목적:
조금 더 적극적인 금액 확인

예:
사정률 100.03%

==================================================
10. 계산 필수값
==================================================

최종 투찰 후보 금액 계산에 반드시 필요한 값:

1. 기초금액
2. 낙찰하한율
3. 적용 사정률

이 3개가 있어야 계산 가능하다.

하나라도 없으면:
“필수 데이터 누락” 상태로 표시한다.

==================================================
11. 최종 계산식
==================================================

예상 투찰 금액 =
기초금액 × 적용 사정률 ÷ 100 × 낙찰하한율 ÷ 100

결과 처리:
- 원 단위 정수 반올림
- 화면에는 천 단위 콤마 표시

예시:
기초금액: 123,456,000원
적용 사정률: 99.98%
낙찰하한율: 87.745%

계산:
123,456,000 × 99.98 ÷ 100 × 87.745 ÷ 100

결과:
108,289,000원

16. 인터넷 검색 및 외부 자료 활용 방향
==================================================

인터넷 검색은 메인 데이터가 아니라 보조 데이터로 사용한다.

메인 데이터:
- 나라장터 입찰공고 API
- 나라장터 낙찰정보 API
- 조달청 공개 데이터

보조 데이터:
- 서울시 계약정보 공개자료
- 각 구청 입찰/계약 공고
- 서울시설공단 입찰 결과
- 서울교통공사 입찰 결과
- 서울주택도시공사 입찰 결과
- 서울시교육청 계약 공개자료

인터넷 검색 활용 목적:
- 누락된 공고 원문 확인
- 기관별 계약 결과 보완
- 과거 입찰 결과 확인
- 특정 공고 PDF 확인

주의:
블로그, 카페, 개인 글은 분석 근거로 사용하지 않는다.
공식 기관 자료만 사용한다.

19. 앱이 하지 말아야 할 것
==================================================

금지 방향:
- 낙찰을 보장한다고 표현하지 않는다.
- AI가 정답을 맞힌다고 표현하지 않는다.
- 비공식 자료를 확정 데이터처럼 사용하지 않는다.
- 공고 조건 확인 없이 자동 투찰을 유도하지 않는다.
- 조달청 원문 확인을 생략하지 않는다.

앱 표현:
“예상”
“시뮬레이션”
“참고용”
“전략 보조”
“과거 유사 공사 기반”

이 표현을 사용한다.

==================================================
20. 최종 앱 정의
==================================================

이 앱은 비드큐를 사용하지 않는다.

최종 정의:
조달청 나라장터 공공데이터 API를 이용해
서울시 관공서의 토목·포장 공사 입찰정보를 실시간으로 수집하고,
과거 유사 공사의 낙찰 결과를 비교 분석하여,
사용자가 조달청 입찰에 입력할 최종 투찰 금액을 결정할 수 있도록 돕는
모바일 카드형 입찰 전략 보조 앱이다.

21. 최종 흐름
==================================================

나라장터 입찰공고 API 연결
→ 서울시 관공서 공사 필터링
→ 토목·포장 관련 공고 필터링
→ 공고 상세정보 확인
→ 유사 공사 낙찰정보 API 검색
→ 과거 낙찰률·사정률 패턴 분석
→ 보수형 / 중간형 / 공격형 투찰가 계산
→ 사용자가 최종 금액 선택
→ 입찰 전 체크리스트 확인
→ 조달청 입찰에 최종 투찰 금액 입력
